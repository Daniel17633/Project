Проект: база данных для системы управления ресторанами

О проекте
Данный проект представляет собой реляционную базу данных для системы управления ресторанами. База данных охватывает ключевые сферы ресторанного бизнеса: управление ресторанами, клиентами, поставщиками, ингредиентами, шеф-поварами, блюдами, столами, заказами, отзывами, а также связями между блюдами и ингредиентами, заказами и блюдами. База данных реализована на системе PostgeSQL.

Структура базы данных

Restaurants - таблица для хранения данных о ресторанах.
Customers - таблица для хранения данных о клиентах.
Suppliers - таблица для хранения данных о поставщиках.
Ingredients - таблица для хранения данных об ингредиентах.
Chefs - таблица для хранения данных о шеф-поварах.
Dishes - таблица для хранения данных о блюдах.
Tables - таблица для хранения данных о столах.
Orders - таблица для хранения данных о заказах.
Reviews - таблица для хранения отзывов.
OrderItems - таблица для хранения элементов заказа (связь между заказами и блюдами).
DishIngredients - таблица для хранения ингредиентов в блюдах (связь между блюдами и ингредиентами).
Связи в базе данных

Restaurants ↔ Chefs. Связь 1:M для связи ресторана с шеф-поварами. 1:M так как в одном ресторане может работать несколько шеф-поваров.
Restaurants ↔ Dishes. Связь 1:M для связи ресторана с блюдами. 1:M так как один ресторан может предлагать несколько блюд.
Restaurants ↔ Tables. Связь 1:M для связи ресторана со столами. 1:M так как в одном ресторане может быть несколько столов.
Restaurants ↔ Orders. Связь 1:M для связи ресторана с заказами. 1:M так как один ресторан может обрабатывать несколько заказов.
Customers ↔ Orders. Связь 1:M для связи клиента с заказами. 1:M так как один клиент может делать несколько заказов.
Customers ↔ Reviews. Связь 1:M для связи клиента с отзывами. 1:M так как один клиент может оставить несколько отзывов.
Suppliers ↔ Ingredients. Связь 1:M для связи поставщика с ингредиентами. 1:M так как один поставщик может поставлять несколько ингредиентов.
Tables ↔ Orders. Связь 1:M для связи стола с заказами. 1:M так как за одним столом может быть несколько заказов (для dine-in).
Orders ↔ Reviews. Связь 1:1 для связи заказа с отзывом. 1:1 так как каждый заказ может иметь только один отзыв.
Orders ↔ OrderItems. Связь 1:M для связи заказа с элементами заказа. 1:M так как один заказ может включать несколько блюд.
Dishes ↔ OrderItems. Связь 1:M для связи блюда с элементами заказа. 1:M так как одно блюдо может быть в нескольких заказах.
Dishes ↔ DishIngredients. Связь 1:M для связи блюда с ингредиентами в блюде. 1:M так как одно блюдо может содержать несколько ингредиентов.
Ingredients ↔ DishIngredients. Связь 1:M для связи ингредиента с блюдами, в которых он используется. 1:M так как один ингредиент может входить в несколько блюд.

Технологии
PostgreSQL
